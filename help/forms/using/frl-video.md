# Script video: Impostazione delle licenze FRL (Feature Restricted Licensing) per Adobe Acrobat su AEM Forms

| Visivo | Titolo | Script altoparlanti |
|--------|-------|---------------|
| Schermata titolo con logo AEM Forms ed elemento grafico FRL | Introduzione | Ti diamo il benvenuto in questo tutorial sulla configurazione di Feature Restricted Licensing, o FRL, per Adobe Acrobat sul tuo server AEM Forms. Questo processo è essenziale in quanto Adobe passa dalle licenze perpetue al nuovo modello FRL. |
| Timeline che mostra le date EOL con frecce che indicano la transizione FRL | Informazioni di base | Le versioni di Adobe Acrobat Classic con licenze perpetue stanno raggiungendo la fine del ciclo di vita, richiedendo una transizione alle versioni più recenti. AEM Forms PDF Generator si basa su Adobe Acrobat per la funzionalità di conversione dei documenti. Il modello Feature Restricted Licensing offre le stesse funzionalità ma con un approccio moderno alle licenze gestito tramite Adobe Admin Console. |
| Elenco di controllo degli elementi richiesti con caselle di controllo e icone | Prerequisiti | Prima di iniziare, assicurati di disporre di tutto il necessario: accesso a Adobe Admin Console con privilegi di amministratore di sistema. In secondo luogo, un account utente con il ruolo di Amministratore della distribuzione in Admin Console. Terzo, privilegi di amministratore locale sul server che esegue AEM Forms. Quarto, un sistema operativo Windows a 64 bit sul server AEM Forms. Quinto, una connessione Internet stabile per l&#39;attivazione della licenza. È inoltre necessario avere familiarità con l’interfaccia di Adobe Admin Console e comprendere l’architettura di distribuzione di AEM Forms. |
| Icona di avviso con elemento grafico di backup | Nota importante | Ricorda di eseguire il backup di eventuali impostazioni Acrobat personalizzate prima di iniziare, in quanto durante questo processo sarà necessario disinstallare le installazioni Acrobat esistenti. |
| Diagramma di flusso che mostra i passaggi numerati da 1 a 7 | Panoramica del processo | Questo processo prevede diversi passaggi chiave: preparazione di un pacchetto FRL in Admin Console, concessione delle autorizzazioni di download, disinstallazione delle versioni precedenti di Acrobat, installazione del nuovo Acrobat Pro, distribuzione del pacchetto FRL e verifica dell’installazione. Passiamo attraverso ogni passaggio. |
| Schermata dell’interfaccia di Adobe Admin Console con scheda Pacchetti evidenziata | Passaggio 1: preparazione di Admin Console | Innanzitutto, accedi a Adobe Admin Console con i privilegi di amministratore di sistema. Passare alla scheda Pacchetti e selezionare &quot;Feature Restricted License&quot; (Licenza con restrizioni per funzioni). Fare clic sul pulsante &quot;Inizia&quot; per iniziare a creare il pacchetto. |
| Schermata Crea configurazione pacchetto con le impostazioni evidenziate | Configurazione pacchetto | Configura il pacchetto con le impostazioni consigliate: seleziona &quot;Offline&quot; per il metodo di attivazione, &quot;PDF Generation&quot; per l’adesione e &quot;Windows 64 bit&quot; per la piattaforma. Per la selezione delle app, conserva solo il file di licenza nelle applicazioni selezionate. Assegna al pacchetto un nome descrittivo come &quot;Acrobat FRL AEM Forms&quot; e crea il pacchetto. |
| Scheda Utenti di Admin Console con finestra di dialogo delle autorizzazioni | Passaggio 2: concedere le autorizzazioni per il download | Successivamente, dovrai creare o identificare un utente che scaricherà il pacchetto. In Admin Console, passa alla scheda Utenti, seleziona l’utente e assegna loro il ruolo di &quot;Amministratore della distribuzione&quot;. Questo consente loro di scaricare il pacchetto FRL creato. |
| Pannello di controllo Campaign Windows con app e funzionalità che mostrano Acrobat | Passaggio 3: disinstallare Acrobat precedente | Prima di installare la nuova versione, è necessario rimuovere completamente tutte le installazioni Acrobat esistenti. Apri il Pannello di controllo Campaign Windows, passa a App, individua Adobe Acrobat e seleziona Disinstalla. Per una rimozione completa, è consigliabile utilizzare lo strumento Adobe Acrobat Cleaner, in particolare se si verificano problemi in un secondo momento. |
| Pagina Download di Adobe Acrobat DC con la sezione del programma di installazione evidenziata | Passaggio 4: Scaricare e installare Adobe Acrobat Pro | Dopo la disinstallazione, scarica il programma di installazione di Acrobat Pro appropriato dalla pagina Download di Adobe Acrobat DC. Per la compatibilità con PDF Generator, si consiglia il programma di installazione a 32 bit. |
| Processo di installazione che mostra i passaggi di estrazione e configurazione | Processo di installazione | Estrarre il file zip scaricato in una cartella, individuare Setup.exe ed eseguirlo come amministratore. Seguire le istruzioni visualizzate sullo schermo per completare l&#39;installazione. Dopo l’installazione, apri Acrobat Pro una volta per chiudere tutte le finestre di dialogo di benvenuto, completando la configurazione iniziale. |
| Scheda Pacchetti Admin Console con il pulsante Scarica evidenziato | Passaggio 5: scaricare il pacchetto FRL | Ora, accedi a Adobe Admin Console utilizzando l’account a cui hai concesso le autorizzazioni di download in precedenza. Passa alla scheda Pacchetti, individua il pacchetto FRL e scaricalo sul server AEM Forms. |
| Prompt dei comandi come amministratore con comandi digitati | Passaggio 6: distribuire il pacchetto | Estrai il pacchetto scaricato in una directory sul server. Apri il prompt dei comandi come amministratore e passa alla directory di estrazione. |
| Riga di comando che mostra il comando di attivazione con sintassi evidenziata | Comando di attivazione | Eseguire il comando di attivazione utilizzando la sintassi seguente: `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Sostituisci il nome file JSON con il nome file esatto del pacchetto. I parametri del comando sono: -p per la piattaforma, -i per installare la licenza e -f per specificare il percorso del file JSON. In caso di esito positivo, verrà visualizzato il messaggio &quot;Operazione completata correttamente&quot;. |
| Tabella che mostra i parametri e le descrizioni dei comandi | Parametri comando | Il parametro -p specifica la piattaforma e rileva automaticamente il sistema operativo. Il parametro -i indica allo strumento di installare e attivare la licenza. Il parametro -f specifica il percorso del file di licenza JSON. |
| Interfaccia di amministrazione di AEM Forms con il servizio PDF Generator | Passaggio 7: Test e verifica | Infine, verifica l’installazione aprendo l’interfaccia di amministrazione di AEM Forms e convertendo un semplice documento in PDF utilizzando il servizio PDF Generator. Inoltre, verifica l’installazione di Acrobat aprendo Acrobat Pro, accedendo alla Guida → Informazioni su e confermando il numero di versione e lo stato della licenza. |
| Schermata divisa: finestra di dialogo Informazioni su Acrobat e conversione PDF riuscita | Verifica | Verifica che la licenza risulti attivata in Acrobat e che PDF Generator sia in grado di convertire correttamente i documenti. |
| Elenco di errori e soluzioni comuni con icone | Suggerimenti per la risoluzione dei problemi | In caso di problemi, verifica questi problemi comuni: assicurati di eseguire il prompt dei comandi come amministratore, verifica che il nome del file JSON sia corretto, verifica la presenza di installazioni in conflitto e verifica che il server disponga della connettività Internet corretta per l’attivazione. |
| Elementi aggiuntivi per la risoluzione dei problemi con segni di spunta | Altre procedure di risoluzione dei problemi | Verifica anche la presenza di restrizioni firewall, pacchetti danneggiati o ID Adobe in conflitto sul server. Se tutto il resto non funziona, contatta il supporto Adobe con messaggi di errore dettagliati. |
| Schermata di successo con icona di segno di spunta e informazioni di contatto | Conclusione | Hai impostato correttamente le Feature Restricted Licensing (Licenze con funzionalità limitate) per Adobe Acrobat sul tuo server AEM Forms. Per ulteriore assistenza, contatta il supporto Adobe o fai riferimento alla documentazione dettagliata disponibile su Experience League. |
| Schermata di chiusura con logo Adobe e crediti | Grazie | Grazie per aver visualizzato questa esercitazione. Per ulteriori informazioni, consulta la documentazione di Experience League. |
